Einführung

Allgemeines zu hashtables

Bei hashtables handelt es sich um Datenstrukturen, die zum schnellen Finden, Einordnen und Löschen von Daten dienen. Bei einer guten Wahl der hash-Funktion (s.u.) dauert ein Suchvorgang bei n vorhandenen Einträgen nur O(1), im Gegensatz dazu bei normalen Listen O(n).

Die Funktionsweise ist wie folgt: Aus jedem Eintrag wird zunächst mittels der sog. Hash-Funktion eine ganze Zahl berechnet, welche in einem bestimmten, endlichen Intervall liegt. Die Hashtable ist nichts anderes als ein Feld, welches als Index die möglichen Werte der Hash-Funktion hat. Jedes Feld-Element ist eine Liste, in der alle zu diesem Hash-Wert gehörigen Einträge stehen.

Will man beispielsweise zufällige ganze Zahlen einsortieren, so könnte man als Hash-Funktion die Funktion "modulo 10" nehmen. Die Zahlen -7, -1, 0, 4, 23, 24, 25, 72, 324 wären in der hashtable dann wie folgt angeordnet:

    ---
   | 0 |  -->   0  -->  300
    ---
   | 1 |
    ---
   | 2 |  -->  72
    ---
   | 3 |  -->  -7  -->  23
    ---
   | 4 |  -->   4  -->   24  -->  324
    ---
   | 5 |  -->  25
    ---
   | 6 |
    ---
   | 7 |
    ---
   | 8 |
    ---
   | 9 |  -->  -1
    ---

Ob die Listen geordnet sind oder nicht, spielt keine Rolle für die Performanz. In dieser Bibliothek handelt es sich um geordnete Listen. Der Nachteil ist, daß man der hashtable auch noch eine Vergleichs-Funktion für die Werte übergeben muß.

Über die Hash-Funktion

Damit eine hashtable optimal funktioniert (d.h. einen optimalen Kompromiß zwischen Speicherplatzverbrauch und Performanz implementiert), muß die Anzahl der möglichen Werte der Hash-Funktion größenordnungsmäßig mit der Anzahl der geplanten Einträge übereinstimmen. Außedem müssen die Werte, die die Hash-Funktion liefert, gleichmäßig über das Intervall verteilt sein.

(Beispielsweise sind, wenn man Menschen einsortieren will, ihr Geburtsjahr (zweistellig), oder der Anfangsbuchstabe ihres Nachnamens ungünstige Hash-Funktionen, da sie überhaupt nicht gleichförmig verteilt sind.)

Interna der Bibliothek

Die Bibliothek benutzt die Listen-Bibliothek lists.h, um ihre Listen zu verwalten. Eine hashtable ist gegeben durch den Typ hashtable_t, wobei man in der Praxis meist mit hashtable_t* zu tun haben wird:

   typedef struct {
     list_t **val;
     int (*cmp_func)(void*, void*);
     long (*hash_func)(void*);
     long hash_min, hash_max;
   } hashtable_t;

Die Bedeutung der einzelnen Elemente dieses strucs ist wie folgt:

val
ist das Feld aus Einträgen vom Typ list_t* wie oben besprochen. val[0] enthält die Einträge mit dem kleinstmöglichen Hash-Wert.
cmp_func
Diese Funktion, die beim Anlegen einer neuen hashtable übergeben werden muß, dient zum Sortieren von Einträgen. Sie soll einen negativen Wert zurückliefern, wenn der erste Eintrag vor dem zweiten kommt, einen positiven, wenn es umgekehrt ist, und Null, wenn die beiden Einträge denselben Inhalt haben. Sie dient zum Sortieren der Listen, und um Gleichheit beim Suchen von Einträgen festzustellen. Aus dem letzten Grund ist es wichtig, daß sie vernünftig implementiert ist, auch wenn man keinen Wert auf sortierte Listen legt.
Läßt sich für die Daten, die man in die hashtable eintragen will, partout keine Ordnung finden, so reicht es hier auch, eine Funktion anzugeben, die im Falle der Gleichheit 0, ansonsten aber immer -1 zurückgibt.
hash_func
Dies ist die Hash-Funktion. Sie muß ebenfalls nur beim Anlegen einer neuen hashtable angegeben werden.
hash_min und hash_max
Dies sind der minimale und der maximale Rückgabewert der Hash-Funktion. Auch sie müssen nur beim Anlegen einer hashtable angegeben werden.

Die Benutzung der Bibliothek

Anlegen und Löschen von hashtables

Bevor man eine hashtable benutzen kann, muß man sie initialisieren. Dabei wird für die Verwaltung der Struktur Speicherplatz belegt, und es wird festgelegt, welche Hash-Funktion und welche Vergleichsfunktion verwendet werden soll. Wird versucht, auf eine nicht initialisierte hashtable zuzugreifen, so gibt es einen Speicherzugriffsfehler.

   hashtable_t *new_hashtable (int (*cmp_func)(void*, void*),
              long (*hash_func)(void*), long hash_min, long hash_max);
   void clear_hashtable (hashtable_t *h, void (*free_func)(void*));

Die Funktion new_hashtable gibt einen Zeiger auf die neue hashtable zurück.

Wird eine hashtable nicht mehr gebraucht, so kann man sie mit der Funktion clear_hashtable löschen lassen. Ist free_func verschieden von NULL, so wird diese Funktion für jeden Eintrag in der hashtable aufgerufen. Sie dient dazu, ev. durch die Einträge belegten Speicherplatz freizugeben.

Einfügen und Löschen von Einträgen

Hierzu dienen die Funktionen

   int insert_hash_element (hashtable_t *h, void *val);
   int remove_hash_entry   (hashtable_t *h, void *val, void (*free_func)(void*));
   int remove_hash_value   (hashtable_t *h, void *val, void (*free_func)(void*));

Die Funktion insert_hash_element fügt den Eintrag val in die hashtable h ein. Sie gibt im Erfolgsfall 0 zurück, ansonsten -1. (Dies kann aber im Augenblick nur auftreten, wenn die Hash-Funktion einen Wert außerhalb der angegebenen Grenzen zurückgibt.)

Die beiden anderen Funktionen dienen zum Entfernen eines Eintrags aus der hashtable h. Vor dem Entfernen wird, sofern verschieden von NULL, die Funktion free_func aufgerufen. Diese dient dazu, ev. durch den Eintrag belegten Speicherplatz wieder freizugeben. Beide Funktionen geben im Erfolgsfall 0 zurück, und -1, falls kein zu löschender Eintrag gefunden wurde.

Der Unterschied zwischen diesen beiden Funktionen ist folgender: Der Funktion remove_hash_entry muß ein tatsächlich in der hashtable vorkommender Zeiger val übergeben werden. Dieser wird dann entfernt. (Sie sucht also nach einem Eintrag v mit v==val.) Die Funktion remove_hash_value sucht in der hashtable einen Eintrag v mit demselben Inhalt, also mit cmp_func(v,val)==0, und entfernt diesen dann.

Finden von Einträgen

   void *find_hash_element (hashtable_t *h, void *val);

überprüft, ob es schon einen Eintrag v in der hashtable gibt, so daß cmp_func(v,val)==0 ist. Falls ein solcher existiert, wird ein Zeiger darauf zurückgegeben. Ansonsten wird ein Nullzeiger zurückgegeben.

Größe einer hashtable feststellen

   long hashtable_size (hashtable_t *h);

Diese Funktion gibt die Gesamtanzahl der Einträge in der übergebenen hashtable zurück.

Umwandeln von hashtables und Listen ineinander, Kopieren von hashtables

Zum Umwandeln von hashtables in andere Datenstrukturen gibt es die folgenden Funktionen:

   void insert_list_into_hashtable (hashtable_t *h, list_t *l, 
                       void* (*copy_func)(void*));
   void insert_hashtable_into_list (list_t *l, hashtable_t *h, 
                       void* (*copy_func)(void*), int sort);
   void insert_hashtable_into_hashtable (hashtable_t *dst, hashtable_t *src, 
                       void* (*copy_func)(void*));

Bei allen drei Funktionen werden die Einträge der Quellstruktur (dem zweiten Argument) in die Zielstruktur (das erste Argument) eingefügt. Die Zielstruktur muß dabei in allen Fällen bereits initialisiert sein. Ob es bereits weitere Einträge enthält, oder ob es leer ist, spielt keine Rolle. Die Quellstruktir wird anschließend nicht(!) gelöscht, sondern besteht weiterhin unverändert fort.

Ist copy_func verschieden von NULL, so werden die Einträge mit Hilfe dieser Funktion kopiert, bevor sie in die Zielstruktur eingetragen werden. Ist sie gleich NULL, so werden die Zeiger so wie sie sind in die Zielstruktur eingefügt.

Der Parameter sort in der Funktion insert_hashtable_into_list gibt an, ob in eine sortierte oder unsortierte Liste eingefügt werden soll. Ist sort!=0, so wird die Liste sortiert.
