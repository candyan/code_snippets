Einführung

rconfig.c ist eine extrem schlanke, aber auch recht minimalistische Bibliothek zum Lesen und Schreiben von Konfigurationsdateien. Sie entstand, weil ich keine Lust hatte, eine 200KB große Datei für so eine läppische Aufgabe einzubinden.

Das Format der Konfigurationsdateien

Die einzelnen Zeilen der Konfigurationsdatei haben folgende Form:

  OPTIONS_NAME = irgendein Text oder andere Zeichen

Der Optionsname, ebenso wie sein Wert kann aus irgendwelchen Zeichen, selbst Leerzeichen oder tabs bstehen. Beide können aber nicht mit Leerzeichen oder tabs beginnen oder enden (s.u.). Ansonsten ist folgendes zu beachten:

  1. Alles, was hinter einem # kommt, wird ignoriert.
  2. In jeder Zeile wird ein = gesucht (dem ersten, welches vorkommt). Kommt kein = vor, wird die Zeile ignoriert.
  3. Die Zeile wird in alles vor dem = und alles nach dem = aufgeteilt. Führende und endende blanks und tabs dieser beiden Teilstrings werden entfernt.
  4. Es wird geschaut, ob eine Option mit dem angegebenen Namen vorhanden ist. Wenn ja, wird ihr Wert auf den zweiten string gesetzt. Wenn nein wird eine Fehlermeldung generiert und weitergemacht.
  5. Zeilen koennen z.Z. nicht mit \ fortgesetzt werden.
  6. Zeilen duerfen nicht laenger als 1024 Zeichen sein.
  7. Es werden keinerlei Sonderzeichen maskiert. Insbesondere können Optionsnamen kein Gleichheitszeichen enthalten.

Lesen und Schreiben von Konfigurationsdateien

Dies geschieht mit den beiden Funktionen

  char *read_config_file  (const char *fname, char **opt_name, char **opt_val);
  char *write_config_file (const char *fname, char **opt_name, char **opt_val,
                           const char *starttext);

Beiden Funktionen wird ein nullterminierter array mit den Options-Namen opt_name übergeben. Der array opt_val enthält die dazugehörigen Options-Werte, welche immer nur char* sein können. Beide Funktionen geben, wenn alles geklappt hat, einen Nullzeiger zurück. Ansonsten geben sie einen Zeiger auf einen dynamisch allokierten String mit einer Fehlermeldung zurück.

read_config_file parst die Datei fname. Wird eine Zeile gefunden, in der eine Option gesetzt wird, so wird der entsprechende Options-Value gesetzt. Ist dieser kein Nullzeiger, so wird der Speicherplatz des alten Wertes nicht(!) freigegeben, und er anschließend neu gesetzt. (Die nicht Freigabe des Speicherplatzes habe ich gemacht, damit man die opt_vals mit konstanten strings vorbelegen kann. Wenn jemand sie mit variablen strings vorbelegen möchte, muß er den Speicherplatz von Hand wieder freigeben.)

write_config_file öffnet ein Datei mit dem angegebenen Namen und schreibt als erstes mal den starttext so wie er ist, hinein. Für das eventuelle Auskommentieren des Textes mit # ist der Anwender selber verantwortlich. Ansonsten schreibt die Funktion für jede Option, deren Wert verschieden von NULL ist, eine Zeile in die Datei.

Zusammensetzen des Namens der Konfigurationsdatei

Es gibt in dieser Bibliothek auch eine Funktion, die einem dabei helfen soll, den Namen der Konfigurationsdatei zu bilden:

char *make_conf_filename (const char *name, const char *default_name, 
                          const char *default_path);

Die Funktion gibt in jedem Fall einen string in einem neu allokierten Speicherbereich zurück. Die alten strings werden unverändert gelassen.

  1. Ist name==NULL, so wird der default_name genommen.
  2. ~ am Anfang wird zum home-directory des Benutzers expandiert.
  3. Ist es kein absoluter Pfad (beginnend mit /), so wird default_path davorkopiert. default_path darf mit / aufhoeren oder nicht.

Beispiele:

make_conf_filename (".config",".default","~");ergibt /home/username/.config
make_conf_filename (NULL,"~/test/.default",NULL); ergibt/home/username/test/.default
